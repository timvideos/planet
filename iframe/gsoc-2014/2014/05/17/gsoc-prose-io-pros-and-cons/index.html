One of needments for TimVideos website is easy and fast interface for edit. Since page will be hosted on GitHub pages there is no way to use one of common CMSes like Wordpress, Radiant CMS or other, we need something that will work with GitHub repository without need for external server.
There are many Markdown editors in the Internet: Markable, Dillinger.io or StackEdit, but they cannot connect with GitHub, they are just editors for Markdown, nothings more (except sharing, saving as a file, preview, etc.)

Fortunately there is an editor with fully integration with GitGub and mostly client-side (needs only very small application on server, because of lack of GitHub authentication).
I would like to use Prose.io with our application, you can check it on http://prose.io, or source on https://github.com/prose/prose

I would like to write some pros and cons for prose.io

Advantages

  • almost fully client-side,
  • full integration with GitHub (you can commit your changes, load your repositories, etc.),
  • easy to maintain and combine with jekyll (check https://github.com/prose/starter),
  • nice editor with preview (and can edit metadata too),
  • build especially for jekyll,
  • you don't need to know git to change your website,
Disadvantages
  • you need to have small application for authentication on server (called gatekeeper),
  • you need to know something about web application to configure jekyll and prose.io,
  • it requires access to all your repositories, both private and public (unless you are running prose.io on your server: http://www.maxmasnick.com/2012/07/03/prose/)
Currently I am still working on design, but I can't wait to combine Prose.io and Jekyll and create easy to maintain website.
