During 4th week I was working on setting our websites to be monitored.

I've done some research, read a lot of articles about monitoring tools and I was so confused about all monitoring tools so I've chosen first from my list; and the choice was great. Monitis.com is very simple monitoring tool that has basic and advanced features, but suits our need in 100%, I've set monitoring for basic HTTP transaction (Monitis checks every one minute from 5-6 destinations if site is alive and if response has text to verify if page exists on requested address), full page load (to monitor time of website loading) and list of transactions (where Monitis plays previously recorded script).

I've added also many small and medium size features to planet.timvideos.us, like:
- social buttons on every post (for Twitter, Facebook and Google+), I am thinking about adding Pocket or other to save post to read it later;
- added excerpt for each post, it's fully configurable in _config.yml, so you can select how many words you want to display in excerpt;
- added Emoji support in post, so you can use Emoji's in your posts, like :1:
- added small tooltip with information about time of reading selected post before you start reading,
- added generate tag, category pages, opml and rss feeds filtered by tags and categories, you can reach them with new RESTfull like routes: ie. planet.timvideo.us/categories/gsoc-2014/tags/news or planet.timvideos.us/tags/github or planet.timvideos.us/categories/gsoc-2014-hdmi2usb (if you want to download RSS or OPML just click on icons that are on the left of category/tag name in top right part of panel;
- fixed bug with wrongly attached layout for some posts,
- small css fixes;
- started working on People part of code.timvideos.us and pulling data from GitHub API

Next week I will work on code.timvideos.us and adding new features for that website and make it awesome!
