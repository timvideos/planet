  • Following a modular approach, the serial expansion board will have a base board and serial adapters.
  • The base board would have 6 x RS232C serial ports with DB9 connectors which would make base board very useful by itself. The DB9 connectors also has excellent mechanical stability for connecting these additional adapters. Now these serial ports could be used directly with VISCA cable or can connect to any other serial device available on internet. I wonder if a serial mouse would work if I connect it to this serial expansion board...
  • These serial ports will lack hardware control because of the way we will be implementing it on the Cypress FX2LP.
  • The serial adapters would be - Serial-GPIO, Serial-RS485. A Serial-DMX adapter can be designed - the protocol is easy, but since a market option is available, will do this only if I run out of things to do. Even RS-232 to RS485 adapters are available to buy, will go through market offerings to see if I need to make one or not.
  • Wondering how I would feed power to the serial-adapters using the base board. I might use the RTS line as explained here, and provide a jumper to disable it.
  • On the other hand, I would like my serial adapters to be usable with other USB-Serial adapters, in case someone wants to use it that way. How should I power it... maybe a 5V DC jack should also be there.
  • To add more value to this board, I'm planing to give complete galvanic isolation instead of ESD chips.
  • Here is a draft specification document for the serial expansion board. Have to enter board design drawings and circuit design. May be, this will end up being the documentation in my repository :)
