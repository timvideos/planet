Last day I wrote about front-end framework choice, I have done some research and work and I have decided to use Twitter Bootstrap.
Yes, it is a large framework with many features that I will not use, but still it is great to build webpages and later on I can remove unused parts of CSS and JS.

Why did I choose Bootstrap?
I have some experience with that framework, I know how to work with it and if I don't there is a great documentation where all my questions have answers.
Bootrap has also great grid system, almost every html component is well styled and it is easy to modify and customize every part of CSS, also there are many components build-in which I can use on website without any special development. I also has full mobile support out of the box, so I can focus on creating the website without thinking about every part of responsive design.

Harish Chouhan wrote great post about Bootrap advantages: http://www.ostraining.com/blog/coding/bootstrap/.
